关于加速器代理规则的说明

在myssh加速器3.2以上的版本里，
用户可以定义哪些网站经过代理哪些不经过。
只需在程序目录(main所在的)下建立rules.txt并写入自定义规则。（linux下是程序目录，win下是lib目录）

在说明规则写法前有几点需要注意的：
1.规则只对加速器服务器有作用，对普通服务器没有作用（因为修改的是加速器客户端）。
2.规则可以适用于http代理或socks代理，但在映射到squid的http代理中无效。
3.规则不分大小写，与浏览器中输入的网址匹配。（例如使用www.163.com和163.com的效果不同。）
4.规则中的分隔符可用一个或多个空格或TAB，但不能使用全角空格。
5.规则生效需要重新连接加速器。

规则写法：
#注释前加半角#号
#使用代理的规则
enable 网址或IP
#不使用代理的规则
disable 网址或IP
#特别的，all会匹配到所有地址
enable all

下面举一个如何使www.163.com不经过代理的例子：
步骤一：在程序目录下建立rules.txt
步骤二：在rules.txt中写入
disable www.163.com
enable all
步骤三：重新连接加速器
加速器在访问网页时会从上到下读取规则，
访问www.163.com时，网址匹配到第一条规则的www.163.com，将会使用第一条规则即不经过代理访问。
访问其他网站时，将会使用第二条规则即通过代理访问。

利用all还可以只让部分网址使用代理，例如：
步骤一：在程序目录下建立rules.txt
步骤二：在rules.txt中写入
enable www.abc.com
disable all
步骤三：重新连接加速器
这时只有www.abc.com会使用代理。
